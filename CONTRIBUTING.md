# Contributing to Loop

Thanks for your interest in Loop. Before you spend time on a contribution, please read this — the short version is **we're not accepting external code contributions yet**.

## Why we're closed for contributions

Loop is in early alpha. Internal architecture, public API surface, build pipeline, and product direction are still moving fast. Reviewing and merging outside PRs against a moving target is overwhelming at this stage

We expect this to change once the project stabilizes — likely after the v1.0 release. When that happens, this document will be rewritten with a real contribution process: setup steps, conventional-commit rules, design-review workflow, and good-first-issue labels.

For now: **please don't open pull requests.** They'll be closed without review, not because we don't appreciate the work, but because we can't review them fairly under our current pace.

## What you *can* do

### Report bugs and request features

[**Open an issue**](https://github.com/abddost/loop/issues/new) — these are reviewed regularly. Helpful reports include:

- The Loop version (Help → About, or visible in the title bar)
- Your OS and architecture (e.g. macOS 15.2 / Apple Silicon)
- Repro steps or a minimal example
- Logs from `~/.local/share/loop/logs/` if the issue involves a crash or the server failing to start
- A screenshot or screen recording for UI issues

Don't post sensitive details (API keys, private repo names, etc.) in issues.

### Report security issues privately

If you find a vulnerability, **do not file a public issue**. Email security@loopai.app (or contact the maintainers privately on GitHub) so we can fix it before details become public.

### Help shape the roadmap

We track feature requests by upvotes and comments. If something matters to you, leave a 👍 on the issue or weigh in on the discussion — we read all of it.

---

Questions about Loop that aren't bugs or feature requests? Reach out via the contact details on [loop-marketing.pages.dev](https://loop-marketing.pages.dev).

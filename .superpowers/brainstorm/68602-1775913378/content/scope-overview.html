How should we approach this?

The ElevenLabs el-* classes and shadow variables are already defined. The question is execution strategy.

A

Bottom-Up: Primitives First (Recommended)

Update UI primitives (button, select, tooltip, snackbar) + global.css classes first. Then refactor components that use them. Changes cascade naturally — fewer total edits needed.

B

Screen-by-Screen: Visual Impact First

Refactor complete screens one at a time (chat page, settings page, sidebar). Immediate visual payoff per screen, but may duplicate work on shared components.

C

Pattern-by-Pattern: One Rule at a Time

Apply one design pattern across ALL components before moving to the next (e.g., all shadows first, then all radii, then all buttons). Systematic but slower visible progress.
